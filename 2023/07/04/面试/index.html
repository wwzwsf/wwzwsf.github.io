HTML

什么是 HTML 语义化，有什么好处

它是一门标记语言，每个标记都有特殊含义，开发者在构建页面时用使用恰当的 HTML 标记。

1.更容易理解，增强了可读性，更方便开发维护。2.在没有 css 样式时也能使页面有清晰的结构。3.可以使搜索引擎能更快速的定位页面中的重要内容。

GET 与 POST 区别

GET 请求用于请求资源，服务器会根据请求参数（放在 URL 中）返回相应的资源，GET 请求的请求参数大小有限制。

POST 请求用于提交数据，将数据打包放在请求体中发送给服务器使用，POST 请求的请求参数大小没有限制。

CSS

盒子居中方式

1：felx 弹性和布局。2、margin:0 auto；3、定位。上下左右居中时子元素用绝对定位然后使用 transform（-50%，-50%）属性将子元素水平垂直居中

px，rem，vw，vh 单位

1、px 是（像素）相对于屏幕分辨率。2、em 参考物是父元素的字体大小。3、rem 根据根元素的大小。4、%相对尺度单位。5、vw 和 vh 根据视窗大小来决定的。

position 的值，作用

relative：相对定位。static：静态定位。sticky：粘性定位。absolute：绝对定位。fixed：固定定位。

JS

路由首位

在 JS 路由中，通常需要使用一个基本路由，即指定页面的根路径，也称为路由首位。例如，基本路由为“/”，那么访问“/about”就是访问“/”下的“about”页面。通过不同的路由参数，可以控制前端页面的导航和操作。

es6

ES6 引入了大量新的语法特性，使得 JavaScript 语言更加强大、灵活和易读。

let 和 const 用于声明变量，它们具有块级作用域，可以防止变量污染和意外覆盖。

Arrow Function（箭头函数）使函数表达式更加简洁。

模板字符串（Template String）可以很方便地构建字符串模板，包括变量插值、多行字符串、处理 HTML 等。

Class（类）可以更容易地创建对象，并支持继承和接口实现等高级特性。

Promise（承诺）可以更优雅地处理异步操作，避免回调地狱。

解构赋值（Destructuring）可以轻松地从数组或对象中获取值，并将它们分配给变量或常量。

其他一些新语法包括 for…of 循环、展开运算符（Spread Operator）、默认参数值（Default Parameter Values）等。

var，const，let 区别

1、var 可以在任何时候重新赋值，也可以在声明之前使用（hoisted）
2、let 可以在任何时候重新赋值，但不能在声明之前使用。
3、const 用于声明一个常量，它具有与 let` 相同的块级作用域，但在声明时必须初始化，且之后不能重新赋值。

防抖与节流

1：防抖(debounce)：在单位时间内频繁触发事件，只有最后一次生效
2：节流(thorttle)：在单位时间内频繁触发事件，只生效一次（也就是只有第一次生效） 1.都可以通过使用 setTimeout 来实现 2.都可以降低回调执行频率，节省计算资源
函数防抖：一段时间内连续触发事件，只执行最后一次
函数节流：一段事件内只执行一次

深拷贝，浅拷贝

1.浅拷贝： 将原对象或原数组的引用直接赋给新对象，新数组，新对象／数组只是原对象的一个引用 2.深拷贝： 创建一个新的对象和数组，将原对象的各项属性的“值”（数组的所有元素）拷贝过来，是“值”而不是“引用”

闭包

闭包是指在一个函数中定义另一个函数，并且内部函数可以访问外部函数的变量，即使外部函数已经执行完毕。在 JS 中，闭包常常用来实现模块化开发和异步编程。例如，在事件处理、网络请求等异步操作中，经常需要使用闭包来保存上下文和状态，确保回调函数能够正确地处理相关数据和事件。

new 操作实例化对象具体做了什么

创建 obj 空对象，继承构造函数的原型对象。让构建函数 this 指向 obj，执行构造函数。如果构造函数返回一个对象，则返回该对象，否则返回刚创建的新对象

事件冒泡

事件冒泡是指当一个元素上的事件被触发时，该事件会向父元素传递，如果父元素也绑定了该事件，则会接着触发父元素上的事件，以此类推直至到达最外层的祖先元素为止。

使用 event.stopPropagation() 方法来阻止事件冒泡。

事件委托

将事件处理程序绑定在某一元素的祖先元素上，通过祖先元素处理该事件，这能够优化性能并减少事件处理程序的数量。

判断一个数组中元素为真

Array.prototype.some()便利每一个数组，返回 true 则至少有一个为真。

Array.prototype.every()返回 true 则每一个都为真。

vuex

Vuex 主要用于集中管理应用程序中的共享组件状态。

Vuex 的核心概念包括：

State：存储所有共享的状态数据。Mutations：用于修改 State 中的数据。Actions：用于解决异步处理逻辑，最终调用 Mutations 修改 State 中的数据。Getters：用于对 State 中的数据进行计算和筛选，返回一个派生状态

Vuex 的工作流程如下：

在 State 中存储应用程序的所有状态。组件通过调用 Mutations 提交一个变更请求。Mutations 更新 State 中的数据。如果组件需要异步请求数据，可以通过 Actions 进行处理。Actions 最终会调用 Mutations 更新 State 中的数据。在需要从 State 中获取数据时，可以通过 Getters 计算派生状态并返回。

生命周期

一个组件实例从创建到销毁过程中，会依次经历一系列的生命周期钩子函数。这些钩子函数可以在组件的生命周期中执行自定义的逻辑代码，方便开发人员进行组件的调试和优化。

挂载阶段：beforeCreate：在实例初始化之后，数据观测之前被调用 0。created：在实例创建完成后被立即调用。beforeMount：在挂载开始之前被调用。mounted：在挂载完成后被调用
更新阶段：beforeUpdate：在数据更新之前被调用，发生在虚拟 DOM 重新渲染和打补丁之前。updated：在数据更新之后被调用，发生在虚拟 DOM 重新渲染和打补丁之后
销毁阶段：beforeDestroy：在实例销毁之前调用。destroyed：在实例销毁之后调用。

件的生命周期钩子函数中，可以执行一系列的操作，例如修改数据、请求数据、注册监听事件、调用父组件方法等等。

尽管有一些生命周期钩子函数可以执行异步操作，但是在 created 钩子函数之前和 destroyed 钩子函数之后，是不能操作 DOM、发送 HTTP 请求等操作的。因此，在开发中需要注意生命周期钩子函数的执行时机和注意前后顺序。

vue2 与 vue3 的区别

Vue.js 3 将 Vue2 的选项式 API 改为组合式 API，这使得组件的封装性更好，简化了数据和逻辑的重用。加强了 TS 的支持。性能得到提升，有更快的渲染速度，包更小。响应式系统也进行了重构，ProxyAPI 提供了更好的性能和更轻的类型推断。

组件通信

在组件通信过程中，需要根据具体情况选择合适的方法，并尽量避免组件之间的耦合过度，以便更好地维护和重用组件。

父子：使用 props 属性可以将父组件的数据传递给子组件。

子父：使用 $emit 方法可以在子组件中触发一个自定义事件，并将数据传递给父组件。

兄弟：使用一个共同的祖先组件作为中介，将数据传递给祖先组件，再通过 props 或事件将数据传递给兄弟组件。

跨组件：使用 provide / inject API 在父组件中提供数据，在子组件中注入数据，实现跨级组件通信。

Vuex 状态管理：当组件之间的数据传递和状态管理变得复杂时，可以使用 Vuex 来集中管理组件之间的数据状态。

v-model 实现原理

双向数据绑定。数据视图绑定，数据驱动试图更新。视图数据绑定，视图驱动数据更新，通过 input 事件获取视图最新值，然后在赋值相应数据。

MVVM

MVVM 为前端应用带来了更好的可维护性、可测试性和解耦。它尤其适用于复杂数字、逻辑复杂的大型应用程序。但对于简单的应用程序来说，可能会导致过度抽象和依赖于特定框架

优点：解耦将 UI 界面与业务逻辑分离，降低了维护成本。双向数据绑定，ViewModel 和 View 数据同步自动完成。其中 ViewModel 可以进行单元测试，其中的代码也可复用。

缺点：过度抽象，内存占用过大，过于依赖框架。

v-bind

把动态的数据绑定到 HTML 元素的属性中。根据应用的状态，动态地更新 UI 页面。它可以用来动态地绑定 class、style、href、src 等属性。

函数组件与类组件区别

语法不同：函数组件是以函数的形式定义的组件，类组件则是以类的形式定义的组件。

继承关系不同：函数组件是无状态组件，不继承任何类，只负责显示内容，不处理状态和生命周期等。React 类组件则是有状态组件，继承自 React.Component 类，可以处理组件状态、生命周期等。

组件性能不同：函数组件相较于类组件有更高的性能，因为函数组件没有状态和生命周期，所以执行起来更快，而且代码更简洁。

Hooks：函数组件可以使用 Hooks，可以通过 Hooks 实现类组件功能，如 useState 这个 Hooks 可以用来处理组件状态，useEffect 可以代替类组件的生命周期函数。

生命周期

挂载：constructor，做一些初始化的工作，更改 this 指向，初始化状态对象，创建业务逻辑。render，渲染视图，不能直接调用 setstart 更新状态。componentDidMount，操作 DOM，发送网络请求，更新组件状态。

更新：componentDidUpdate: 操作 DOM、发送网络请求、有条件的更新组件状态。

卸载：componentWillUnMount 清理操作、清除定时器、清除事件、清除订阅、清除全局状态。

Hooks

是 v6 在函数组件中使用状态和其他 React 特性而不需要写成类组件。

简化代码：使用 Hooks 可以将业务逻辑拆分成多个自定义 Hook。提高可读性和可维护性：组件功能更加清晰，更容易测试和维护。提高性能：Hooks 结合了函数式编程的优势，在有状态的组件中可以避免不必要的渲染，提高应用性能。

useState：该 Hook 用于在函数组件中声明状态，并提供了更新该状态的方法。

useEffect：用于在函数组件中执行副作用（如获取数据、订阅事件等）或处理组件的生命周期（如组件挂载、更新、卸载等）。

useContext：用于在函数组件中访问全局的上下文（Context）数据。

useReducer：用于在函数组件中使用 Redux 中的 reducer 函数实现状态管理。

useCallback：用于在函数组件中进行性能优化，避免组件重复渲染。

在使用 Hooks 时，一定要保证组件本身无副作用，否则会影响应用性能和可维护性。同时，也要注意 Hooks 的使用规则，避免出现类似闭包导致的问题。

useEffect 两个参数

参数一：一个函数，需要执行的副作用操作，要发送的网络请求等

参数二：一个数组，副作用代码的依赖项，状态或属性发生变化副作用代码重新执行可以很好地控制 useEffect 的执行时机和条件，从而避免不必要的性能问题。若为空数组则副作用操作只在初次渲染时操作

Redux

React是用于构建用户界面的，帮助我们解决渲染DOM的过程

而在整个应用中会存在很多个组件，每个组件的state是由自身进行管理，包括组件定义自身的state、组件之间的通信通过props传递、使用Context实现数据共享

如果让每个组件都存储自身相关的状态，理论上来讲不会影响应用的运行，但在开发及后续维护阶段，我们将花费大量精力去查询状态的变化过程

这种情况下，如果将所有的状态进行集中管理，当需要更新状态的时候，仅需要对这个管理集中处理，而不用去关心状态是如何分发到每一个组件内部的

store：对象进行状态的管理，store 对象可以存储状态，有提供了管理状态的方法。

reducer：用于接收调用 dispatch 方法接收 action 对象，视图中要想修改状态。

action：对象进行描述，action 对象中有 type 属性，它就是具体的用来描述的属性。

Router

path 属性：用于设置匹配到的路径。component 属性：设置匹配到路径后，渲染的组件。render 属性：设置匹配到路径后，渲染的内容。exact 属性：开启精准匹配，只有精准匹配到完全一致的路径，才会渲染对应的组件

swich 组件：作用适用于当匹配到第一个组件的时候，后面的组件就不应该继续匹配

懒加载：通过 React.lazy（）和 React.Suspense（不问就知说 lazy） 组件实现

路由守卫：路由的匹配顺序是从上到下逐个匹配，在匹配到第一个符合要求的路由后立即跳转，

两种模式：React Router 对应的 hash 模式和 history 模式对应的组件为：HashRouter，BrowserRouter
